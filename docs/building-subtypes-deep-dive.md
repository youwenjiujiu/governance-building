# Building Sub-Types 深度拆解

> 同一个大类下的楼宇，因为定位、规模、业态的不同，运营模式差异巨大。
> 本文按每种楼宇大类，穷举其子类型，并标注运营差异点。

---

## 1. 酒店（Hotel）

### 子类型对比

| 子类型 | 规模 | 客房数 | 系统复杂度 | 运营模式 |
|--------|------|--------|-----------|---------|
| **经济型/快捷酒店** | 小 | 80-200 | 低 | 极致标准化，最少人工 |
| **商务酒店** | 中 | 150-400 | 中 | 会议设施为核心卖点 |
| **全服务五星酒店** | 大 | 300-800 | 高 | 全系统配齐，服务为王 |
| **度假酒店/Resort** | 大 | 200-500 | 高 | 户外设施多，季节性强 |
| **精品/设计酒店** | 小 | 30-100 | 中 | 个性化极强，非标准化 |
| **公寓式酒店/Serviced Apt** | 中 | 100-300 | 中 | 长住为主，类住宅管理 |
| **民宿/客栈** | 极小 | 5-30 | 低 | 老板即运营，智能化需求简单 |
| **胶囊旅馆/青旅** | 小 | 50-200床位 | 低 | 公共区域为主，隐私空间极小 |
| **会议型酒店/Convention Hotel** | 大 | 500-2000 | 极高 | 大宴会厅/分会场是核心资产 |
| **赌场酒店/Casino Hotel** | 超大 | 1000-6000 | 极高 | 赌场区域24h满负荷 |

### 运营差异详解

#### 经济型/快捷酒店（如家/7天/汉庭/OYO）

```
特点：
- 无餐厅或仅早餐
- 无泳池/健身房/宴会厅
- 客房控制极简：分体空调 + 取电开关 + 基本照明
- 无 GRMS，空调由客人手动操作
- 公区面积小，主要是前台和走廊
- 连锁品牌强调标准化复制

运营核心：
- 能耗成本控制（利润薄，能耗占比高）
- 空房自动关空调（取电开关实现）
- 热水系统是最大挑战（经济型通常用空气源热泵）
- 保洁排班自动化
- 远程集中管理（一个区域经理管 5-10 家店）

产品设计启示：
→ 需要「连锁集中管控」模式 — 总部看所有门店的能耗/告警
→ 设备配置极简，但要批量管理（100家店 × 150间房 = 15000 个节点）
→ 自助入住（减少前台人力）→ 门锁/PMS 集成必须有
```

#### 全服务五星酒店（万豪/希尔顿/香格里拉）

```
特点：
- 完整 GRMS（客房管理系统）
- 多餐厅/酒吧（不同风格）
- 泳池/SPA/健身中心
- 大宴会厅（可分隔）+ 多个小会议室
- 行政酒廊/俱乐部楼层
- 地下车库/代客泊车
- 洗衣房（工业级）
- 后勤区域（员工餐厅/更衣室/仓库）

运营核心：
- VIP 客人偏好管理（CRM → GRMS 联动）
- 客房状态精细管理（入住/退房/维修/深度清洁）
- 能源管理（五星酒店 EUI 可达 300-500 kWh/m²/年）
- 热水系统可靠性（365天 × 24小时不间断）
- 大宴会按活动切换（婚宴/会议/晚宴，灯光/温度/隔断全变）
- 品牌标准合规（每个品牌有严格的温度/照明/空气质量标准）

产品设计启示：
→ GRMS 深度集成是核心卖点
→ 需要「活动模式」引擎 — 按预约自动切换宴会厅/会议室环境
→ 品牌标准模板（万豪标准/希尔顿标准 → 自动合规检查）
→ 后厨系统（HACCP 温度记录）是法规刚需
```

#### 度假酒店/Resort

```
特点：
- 大面积室外区域（花园/泳池/沙滩/高尔夫）
- 多栋建筑分散布局（非单体高层）
- 别墅/独栋客房
- 室外活动设施（水上乐园/儿童乐园/运动场）
- 强季节性（旺季满房/淡季空置）

运营核心：
- 室外设施管理（泳池水质、景观灌溉、室外照明）
- 别墅区分散管理（每栋独立空调/热水/安防）
- 季节性能耗差异极大
- 淡季设备维保（利用空闲期大修）
- 虫害/园林养护

产品设计启示：
→ 需要「分散式多建筑」管理模式（一个酒店 = 多个 Building）
→ 室外系统监控（灌溉/泳池/照明）是刚需
→ 季节模式切换（旺季全开/淡季部分关闭/维保模式）
```

#### 赌场酒店/Casino Hotel

```
特点：
- 赌场区域 24h 营业、全年无休
- 赌场无窗无钟（刻意模糊时间感）
- 极强的空调需求（人密+设备热+烟雾处理）
- 超大面积（赌场可达数万平米）
- 高安全等级（海量摄像头、反作弊、现金管理）
- 娱乐/演出/餐饮/购物综合体

运营核心：
- 赌场区空气质量（烟雾处理、新风量极大）
- 7×24 不间断运行的空调/照明/安防
- 高密度视频监控（赌桌/筹码/现金区/VIP房）
- 电力高可靠性（赌场停电 = 法规问题）
- 氛围控制（温度偏低保持清醒、照明特殊设计）

产品设计启示：
→ 需要「永不停机」模式 — 维护窗口极小
→ 安防深度集成是刚需（不只是门禁，是反欺诈级别）
→ 能耗巨大但不敏感（赌场不差钱，但合规要求高）
```

## 2. 医院（Hospital）

### 子类型对比

| 子类型 | 床位 | 系统复杂度 | 核心特征 |
|--------|------|-----------|---------|
| **社区卫生中心/诊所** | 0-20 | 低 | 门诊为主，无住院，简单空调照明 |
| **二级综合医院** | 100-500 | 中 | 基本科室齐全，手术室有限 |
| **三甲综合医院** | 500-3000 | 极高 | 全科室+高等级洁净+完整医气+科研 |
| **专科医院-肿瘤** | 200-1000 | 高 | 放疗加速器（辐射防护）、化疗配液（洁净） |
| **专科医院-儿童** | 200-800 | 高 | 温度偏高、安全防护（防坠/防夹）、家属陪住 |
| **专科医院-传染病** | 100-500 | 极高 | 全院负压设计、BSL-3 实验室、隔离流线 |
| **专科医院-精神科** | 100-500 | 中 | 防自伤设计、封闭管理、安防重于医疗设备 |
| **专科医院-口腔/眼科** | 50-200 | 中低 | 门诊为主，小型手术室 |
| **康复医院** | 100-500 | 中 | 康复训练设施、无障碍要求极高、温水泳池 |
| **妇产医院** | 100-500 | 高 | 产房/NICU/月子中心、新生儿区域特殊温控 |
| **中医医院** | 100-500 | 中 | 煎药房（排气）、针灸/推拿（温度要求）、药材仓储（防潮防虫） |
| **军队医院/野战医院** | 不定 | 特殊 | 战备要求、模块化/可移动、自给能源 |

### 运营差异详解

#### 三甲综合医院

```
特点：
- 科室 30-50 个，建筑面积 10-30 万平米
- 手术室 20-50 间（百级/千级/万级分级）
- ICU/CCU/NICU/PICU/EICU 多个重症单元
- 完整医气系统（O2/真空/压缩空气/N2O/CO2/N2）
- 核医学科/放射科（辐射防护）
- 中心实验室（BSL-2/3）
- 中心供应室（CSSD，器械清洗灭菌）
- 血库/药库/冷库体系
- 气动物流系统（全院管道传输）
- 日间手术中心
- 教学/科研区域
- 太平间

运营核心：
- 24/7 不间断运行（急诊/ICU/手术室/医气/冷库）
- 手术室排程联动（空调提前/术后换气/消毒周期）
- 院感防控（负压隔离/空气培养/温湿度合规记录）
- 医气 365 天零故障（供氧中断 = 生命危险）
- 冷链 100% 不断链（血液/疫苗/样本）
- 停电 10 秒内切换（手术中/呼吸机/监护仪）
- 合规记录海量（温度/湿度/压差/医气压力/消毒/冷链 全要留痕）

产品设计启示：
→ 合规引擎是核心 — 自动生成温湿度/医气/冷链的合规报表
→ 需要「科室级」管理粒度（手术室和门诊完全不同的运营逻辑）
→ 医气监控是独立模块（其他楼宇类型没有）
→ 与 HIS/手术排程 的深度集成
```

#### 传染病医院

```
特点：
- 全院或大部分区域负压设计
- 严格的清洁区/半污染区/污染区三区划分
- 患者通道和医护通道完全分离
- BSL-3 甚至 BSL-4 实验室
- 污水全部预消毒处理
- 医废特殊处理流程
- 空气经 HEPA 过滤后高空排放

运营核心：
- 压差梯度监控（走廊→缓冲→病房，每级 -5Pa 到 -15Pa）
- 排风 HEPA 滤网更换周期管理
- 患者追踪（RFID，防止离开隔离区）
- 医护防护用品消耗监控
- 污水处理在线监测（消毒剂余量/细菌培养）
- 尸体转运特殊流程

产品设计启示：
→ 压差监控是第一优先级（压差反转 = 病原扩散）
→ 需要「三区」空间模型 — 每个区域的通风/门禁规则不同
→ 废水/废物追踪必须完整闭环
```

#### 精神科医院

```
特点：
- 封闭管理区域（患者不能自由出入）
- 防自伤设计（无凸出物/隐藏管道/安全门把手/防碎玻璃）
- 无法使用传统门禁卡（患者不能持有）
- 活动室/治疗室/园艺治疗区
- 药品严格管控（尤其管制精神类药物）
- 约束/隔离室

运营核心：
- 安防重于一切（防逃跑/防自伤/防攻击）
- 门禁由护士站集中控制（患者区域所有门）
- 视频 AI：检测异常行为（自伤动作/攻击/试图翻越）
- 室内环境尽量温馨（非监狱感，但安全第一）
- 户外放风区域围界
- 特殊用药记录与环境监控关联

产品设计启示：
→ 安防逻辑完全不同于其他医院 — 更接近监狱但要"温和"
→ 门禁不是刷卡模式，是中央控制模式
→ AI 行为检测是差异化功能
```

## 3. 商场/零售（Retail）

### 子类型对比

| 子类型 | 面积 | 租户数 | 系统复杂度 | 核心特征 |
|--------|------|--------|-----------|---------|
| **社区商业/底商** | 0.5-2万m² | 10-50 | 低 | 服务周边社区，简单管理 |
| **购物中心/Mall** | 5-15万m² | 100-300 | 高 | 中庭/影院/餐饮/娱乐综合 |
| **超大型商业综合体** | 15-50万m² | 300-800 | 极高 | 购物+办公+酒店+公寓+展览 |
| **奥特莱斯/Outlets** | 3-10万m² | 50-200 | 中 | 开放式街区，室外为主 |
| **百货商场** | 2-8万m² | 自营+联营 | 中 | 单一业主，统一管理 |
| **家居建材城** | 3-20万m² | 100-500 | 中 | 大件货品/仓储式/叉车动线 |
| **批发市场** | 5-30万m² | 500-5000 | 中 | 档口密集、消防重点、人货混行 |
| **商业步行街** | 线性分布 | 50-300 | 中 | 室外+室内混合，市政设施 |
| **免税店/机场商业** | 0.5-5万m² | 50-200 | 高 | 航班联动/24h/安检限制 |

### 运营差异详解

#### 购物中心/Mall

```
特点：
- 有中庭（采光/活动/装饰核心空间）
- 影院/KTV/健身房/冰场等主力店
- 餐饮占比 30-40%（油烟处理是大挑战）
- 停车场（地下1-3层，500-3000车位）
- 高客流波动（工作日vs周末差3倍，节假日更极端）

运营核心：
- 租户计量计费（电/水/空调 — 分户计量是物业收入基础）
- 公区vs租户的费用分摊（公摊算法很复杂）
- 餐饮区排油烟（统一排烟管道，按排放量收费）
- 客流数据驱动运营（哪层人多、哪个入口有效、驻留时间）
- 主力店 vs 中小租户的服务差异
- 装修期间管理（新租户进场的施工管理）
- 中庭活动（搭台/拆台的临时用电/空调/安防）

产品设计启示：
→ 「租户计量计费」是核心模块 — 直接涉及物业收入
→ 客流分析集成很有价值（能耗 vs 客流 关联分析）
→ 需要「租户装修管理」工作流（审批/施工/验收/转正式）
```

#### 超大型商业综合体（如万达广场/华润万象城）

```
特点：
- 多业态：零售 + 写字楼 + 酒店 + 公寓 + 展览中心
- 多栋建筑，地下连通
- 可能有独立冷站/热站（区域能源中心）
- 多个物业管理公司（商业/办公/住宅 分开管理）
- 多套独立系统（每栋楼自己的 BMS，需要统一平台）

运营核心：
- 跨业态能源调度（商场白天用冷多，酒店晚上用热多 → 蓄能互补）
- 多套 BMS 统一监控（不同厂商的系统要集成到一个平台）
- 共享设施管理（地下车库/冷站/变配电 多方共用）
- 费用分摊极其复杂（按面积/按冷量/按实际用量 不同算法）

产品设计启示：
→ 需要「多建筑/多业态」管理架构
→ 多 BMS 网关集成能力（同一个园区有西门子+霍尼韦尔+江森的系统）
→ 费用分摊计算引擎（可配置多种算法）
```

#### 批发市场

```
特点：
- 档口极多（可达数千个），每个 5-20 m²
- 电气安全是最大隐患（私拉电线、超负荷）
- 消防是重中之重（货品密集、通道窄、人流杂）
- 物流通道（货车进出、叉车、推车）
- 冷链区域（水产/肉类/冷冻食品）

运营核心：
- 消防监控密度极高（每个档口烟感）
- 用电安全监控（电弧检测、温度监测、超负荷告警）
- 冷链区温度不间断监控
- 人货混行安全管理
- 营业时间外的安防（深夜货物安全）

产品设计启示：
→ 消防 + 电气安全 是第一优先级（比舒适度重要得多）
→ 需要支持「极高密度点位」—— 5000 个档口 × 5 个监测点 = 25000 个点位
→ 冷链合规记录（食品安全法规）
```

## 4. 写字楼/办公（Office）

### 子类型对比

| 子类型 | 面积 | 系统复杂度 | 核心特征 |
|--------|------|-----------|---------|
| **甲级写字楼** | 5-15万m² | 高 | 高标准 MEP，租户多为大企业 |
| **乙级写字楼** | 2-8万m² | 中 | 标准配置，中小企业 |
| **联合办公/WeWork** | 0.2-2万m² | 中 | 灵活工位，按座位/天计费 |
| **产业园区办公** | 园区多栋 | 中 | 研发+办公，配套食堂/运动设施 |
| **政府办公楼** | 1-10万m² | 中高 | 安全等级高，保密要求，节能考核 |
| **总部大楼/Headquarters** | 5-30万m² | 极高 | 企业形象，智能化标杆 |
| **SOHO/小型办公** | 0.1-0.5万m² | 低 | 商住混合，管理粗放 |

### 运营差异详解

#### 联合办公/WeWork 模式

```
特点：
- 不是按楼层/房间出租，是按工位/会议室/时段出租
- 高度灵活：今天 20 人用、明天 50 人用
- 公共区域大（休息区/咖啡区/电话亭/活动区）
- 会议室按小时预订、频繁换人
- 租户变化频繁（月租/季租为主）

运营核心：
- 工位占用实时检测 → 空调照明按实际占用动态调整
- 会议室高频联动（一天 10 场会议，每场切换环境）
- 空间利用率优化（哪些区域长期空置 → 调整布局）
- 按人头/工位/时间的灵活计费
- 访客频繁 → 门禁要灵活但安全

产品设计启示：
→ 「按占用调节」是核心策略 — 不是固定时间表，是实时响应
→ 空间利用率分析是对运营方的核心价值
→ 计费模型要支持「按工位·天」「按会议室·小时」
```

#### 政府办公楼

```
特点：
- 安全分区（对外窗口/内部办公/领导区/机要室）
- 保密要求（网络隔离/电磁屏蔽会议室）
- 信访接待区（独立动线、安保加强）
- 应急指挥中心（视频墙/通信/GIS）
- 档案室（恒温恒湿/防火/防潮/防虫）
- 节能考核（公共机构节能条例）

运营核心：
- 公共机构能耗限额管理（能耗超标 = 考核扣分）
- 安全分区门禁（不同级别不同权限）
- 信访高峰期应急预案
- 档案室环境恒定（温度 14-24°C、湿度 45-60%）
- 领导用车/公务用车管理
- 对外形象管理（大厅/外立面/会议室高标准）

产品设计启示：
→ 节能考核报表是硬需求（直接对标国家公共机构节能标准）
→ 安全分区逻辑比普通写字楼复杂得多
→ 保密/信创要求 → 可能需要国产化部署方案
```

### 运营差异详解

#### 综合性大学

```
特点：
- 多校区（可能跨城市）
- 建筑类型极多：教学楼/科研楼/实验室/图书馆/体育馆/食堂/宿舍/行政楼/医院

- 节假日/寒暑假 空置率极高

运营核心：
- 课表/学期联动（上课 → 开空调照明，假期 → 全部关）
- 科研楼 24h 运行（实验不停，但公共区域分时管理）
- 多类型建筑统一管理（教学楼和实验室的运营逻辑完全不同）
- 宿舍限电（大功率跳闸、夜间断电）
- 校园能耗大户管理（游泳馆/数据中心/实验室）
- 新能源示范项目（光伏/储能，大学喜欢做示范）

产品设计启示：
→ 大学 = 一个微缩城市，几乎涵盖所有楼宇类型
→ 课表/学期日历是核心调度依据
→ 需要支持「建筑群」管理（一个大学 = 50+ 栋楼）
→ 实验室安全管理是高价值场景
```

#### 幼儿园

```
特点：
- 极其关注安全（防走失/防拐/防坠落/防夹伤）
- 家长接送管理（人脸识别 + 授权人匹配）
- 室内温度偏高（幼儿体温调节能力弱，冬季 22-24°C）
- 空气质量高要求（甲醛/TVOC/PM2.5）
- 厨房明厨亮灶（家长可远程查看）
- 教室/午睡室 温湿度精细管理
- 户外活动区安全监控

运营核心：
- 接送管理是核心安全场景（只有授权人才能接走孩子）
- 室内空气质量是家长第一关注点
- 厨房食品安全（HACCP + 明厨亮灶直播）
- 午睡时段 vs 活动时段 环境切换
- 消毒记录（疫情后家长非常关注）

产品设计启示：
→ 家长端 APP 是必须有的（看到空气质量/厨房直播/接送记录）
→ 接送管理 = 特殊门禁场景（不是卡/指纹，是人脸+授权关系）
→ 空气质量实时展示（大屏 + APP）是招生卖点
```

### 运营差异详解

#### 音乐厅/剧院

```
特点：
- 声学环境极其严格（混响时间 RT60 精确控制）
- 空调必须超低噪音（NC15-NC20，比住宅安静得多）
- 温湿度精密控制（乐器对温湿度敏感，尤其钢琴/管风琴）
- 舞台机械（升降台/旋转台/吊杆/防火幕）
- 专业舞台灯光（大量热负荷）
- 观众厅照明（演出时极暗/幕间微亮/散场全亮）

运营核心：
- 演出前 2h：空调低噪音模式→温度预冷（观众入场后体温升高）
- 演出中：空调最低噪音→灯光由演出方控制→安全监控
- 幕间：提升空调→照明微亮→允许出入
- 散场：全亮→空调维持→清场
- 乐器存放区恒温恒湿（钢琴间 20-22°C，45-55%RH）
- 舞台灯光的热负荷管理（数百kW 灯光→巨大冷量需求）

产品设计启示：
→ 「演出模式引擎」— 按节目单自动切换
→ 噪音监测是独特需求（空调噪音直接影响演出质量）
→ 温湿度对乐器的影响 → 需要精密记录和告警
```

#### 主题公园

```
特点：
- 园区面积巨大（数十到数百公顷）
- 设施类型极多：
  - 游乐设备（过山车/摩天轮/旋转木马/漂流）
  - 室内项目（4D 影院/鬼屋/密室）
  - 演出场所（剧场/花车巡游/烟花秀）
  - 餐饮（几十个餐厅/小食亭）
  - 零售（纪念品商店）
  - 酒店（园区内度假酒店）
  - 后勤（维修车间/员工区/仓库/垃圾处理）

运营核心：
- 游乐设备安全监控（运行状态/维保周期/天气限制）
- 大客流管理（入园人数/区域人数/排队时间/动态引导）
- 室外环境管理（喷雾降温/遮阳/防雨/防风/防雷）
- 水处理（漂流/水上项目的水质）
- 烟花秀/巡游的全系统联动（灯光/音响/安防/交通管制）
- 恶劣天气应急（高空项目停运/室外项目关闭/疏散）

产品设计启示：
→ 主题公园 = 一个小型城市，是最复杂的楼宇运营场景
→ 游乐设备安全监控是独特的垂直场景
→ 客流+天气 驱动的动态运营模式
```

## 8. 工业/制造（Industrial）

### 子类型对比

| 子类型 | 系统复杂度 | 核心特征 |
|--------|-----------|---------|
| **电子制造（SMT/封装）** | 极高 | 洁净室 Class 100-10000，恒温恒湿 ±0.5°C |
| **半导体 Fab** | 顶级 | 洁净 Class 1-10，振动/EMI控制，超纯水/特气 |
| **制药 GMP 工厂** | 极高 | A-D 级洁净，验证体系，全程记录 |
| **食品加工厂** | 高 | 冷链/卫生/HACCP/虫害/排放 |
| **汽车/机械加工** | 中高 | 焊接烟尘/喷涂VOC/大型天车/压缩空气 |
| **纺织/服装** | 中 | 粉尘/噪音/恒温恒湿（纺织品质量） |
| **化工厂** | 极高 | 防爆/有毒气体/废气废水/SIL安全 |
| **电池工厂（锂电/PACK）** | 极高 | 干燥间（露点-40°C）/防爆/消防特殊 |
| **印刷/包装** | 中 | VOC排放/恒温恒湿（套印精度）/粉尘 |
| **酿酒/发酵** | 中高 | 温度精确控制/CO2排放/清洗CIP |

### 运营差异详解

#### 半导体 Fab

```
特点：
- 洁净度要求全球最高（Class 1-10，即每立方英尺 <10 个 0.5μm 颗粒）
- 恒温恒湿精度 ±0.1°C / ±1%RH
- 振动控制（VC-A 到 VC-G 级，纳米级精度设备）
- 电磁干扰（EMI）控制
- 超纯水系统（电阻率 18.2 MΩ·cm）
- 特种气体系统（数十种，含剧毒气体如 AsH3, PH3）
- 废气处理（酸性/碱性/有机/特气分类处理）
- 废水处理（含氟/含铜/含有机物分类处理）
- 24/7/365 不间断生产
- 单次停机损失可达千万美元

运营核心：
- 洁净度是生命线（一个颗粒 = 一片报废晶圆）
- 特气安全（微量泄漏即致命，双层管道+连续检测）
- 振动监测（附近施工/交通都可能影响）
- 超纯水水质在线监测（电导率/TOC/颗粒/细菌）
- 停机计划精确到小时（设备维护窗口极窄）

产品设计启示：
→ 这是工业级最高要求的场景
→ 环境参数精度要求远超普通楼宇（±0.1°C vs ±1°C）
→ 特气安全是独有场景（其他行业没有）
→ 停机成本极高 → 预测性维护价值最大
```

#### 锂电池工厂

```
特点：
- 干燥间（Dry Room）— 露点 -40°C 到 -60°C（极端干燥）
- 锂电池火灾风险极高（热失控/链式反应）
- 传统灭火剂（水/干粉）对锂电池火灾效果差
- NMP 溶剂回收（有毒/易燃）
- 粉尘管理（正极/负极材料粉尘）
- 产线洁净度要求（虽不如半导体，但仍有要求）

运营核心：
- 干燥间露点监控（露点升高 = 产品质量问题）
- 热失控预警（红外热成像/气体检测/烟雾检测）
- 特殊消防系统（大量水喷淋+持续冷却，非传统灭火思路）
- NMP 泄漏检测与排放控制
- 储能区域安全（成品电池存储的温控和消防）

产品设计启示：
→ 锂电池消防是全新领域（与传统消防逻辑不同）
→ 干燥间是独有环境类型
→ 热失控预警 → AI/红外是差异化
```

```
特点：
- 单柜功率密度极高（传统 5-8kW/柜 vs AI 30-100kW/柜）
- 传统风冷不够，必须液冷（直接芯片液冷/浸没式液冷）
- 电力需求巨大（单集群可达数十 MW）
- 散热量巨大（几乎所有电力都变成热量）
- 设备价值极高（一台 GPU 服务器 >100 万元）
- 连续运行数周（训练任务不能中断）

运营核心：
- 液冷系统管理（冷却液温度/流量/压力/泄漏）
- GPU 温度实时监控（过热降频 = 训练减速 = 直接损失）
- 电力保障（训练中断 = 数周工作白费）
- PUE 极致优化（液冷 PUE 可达 1.05-1.1）
- 冷却液维护（定期检测/更换/补充）

产品设计启示：
→ 液冷监控是全新子系统（传统 BMS 不覆盖）
→ GPU 温度/功耗与冷却系统的闭环控制
→ 训练任务 vs 基础设施的关联分析
```

## 总结：楼宇子类型的关键认知

### 1. 复杂度光谱

```
最简单 ←──────────────────────────────────→ 最复杂

民宿 → 快捷酒店 → 社区商业 → 普通住宅 → 写字楼 → 商场 → 五星酒店
→ 大学 → 工厂 → 医院 → 数据中心 → 半导体Fab → 主题公园
```

### 2. 产品分层策略

| 层级 | 适用楼宇 | 功能范围 | 复杂度 |
|------|---------|---------|--------|
| **基础版** | 住宅/社区商业/快捷酒店/小型写字楼 | 监控+告警+基础能耗 | 低 |
| **标准版** | 商场/写字楼/学校/中型酒店 | + 租户计量 + 工单 + 策略引擎 | 中 |
| **专业版** | 五星酒店/医院/工厂/数据中心 | + 行业专属模块 + 深度集成 | 高 |
| **企业版** | 大型综合体/连锁/集团 | + 多楼宇管理 + 对标分析 + 开放平台 | 极高 |

### 3. 行业专属模块清单

| 楼宇类型 | 必须有的专属模块 |
|---------|----------------|
| 酒店 | GRMS 客房控制、PMS 集成、热水系统、泳池水质 |
| 医院 | 医气监控、洁净度管理、冷链追踪、院感防控、HIS 集成 |
| 商场 | 租户计量计费、客流分析、排油烟监测、租户装修管理 |
| 数据中心 | PUE 管理、精密空调、液冷系统、容量管理 |
| 工厂 | 洁净室、工艺环境、气体检测、MES 集成、废气废水 |
| 学校 | 课表联动、宿舍限电、实验室安全、明厨亮灶 |
| 警局/监狱 | 高安全门禁、审讯/监控录像、人员定位、武器/证据管理 |
| 养老院 | 跌倒/离床检测、防走失、护理呼叫、健康监测 |
| 博物馆 | 精密温湿度、光照控制、文物防盗 |
| 场馆 | 场地模式切换、赛事联动、大客流管理 |
