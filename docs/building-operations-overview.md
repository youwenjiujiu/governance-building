# Building Operations 全景分析（完整版）

> 目标：穷举所有楼宇类型的所有运营场景，确保产品设计不遗漏任何一个维度。

---

## 一、楼宇的生命周期阶段

| 阶段 | 英文 | 运营关注点 |
|------|------|-----------|
| 设计 | Design | 系统选型、点位规划、能耗模拟 |
| 施工调试 | Commissioning | 设备安装验证、系统联调、参数初调 |
| 交付验收 | Handover | 竣工资料归档、培训、基线建立 |
| 日常运营 | Operations | **核心阶段** — 本文档重点 |
| 改造升级 | Retrofit | 系统升级、节能改造、IoT 加装 |
| 退役拆除 | Decommission | 设备回收、数据迁移 |

## 二、通用子系统（所有楼宇都有）

以下系统几乎存在于每一种楼宇中，是基础底座：

### A. 暖通空调（HVAC）

```
冷热源
├── 冷水机组（Chiller）
├── 热泵（Heat Pump）
├── 锅炉（Boiler）
├── 冷却塔（Cooling Tower）
└── 蓄冷/蓄热（Thermal Storage）

输配系统
├── 冷冻水泵（Chilled Water Pump）
├── 冷却水泵（Condenser Water Pump）
├── 热水循环泵（Hot Water Pump）
├── 管道阀门（Valve）
└── 分集水器（Header）

末端设备
├── 空气处理机组（AHU）
├── 新风机组（MAU/DOAS）
├── 风机盘管（FCU）
├── VAV 变风量末端
├── 分体空调（Split AC）
└── 地暖/散热器

控制与传感
├── 温度传感器 / 湿度传感器
├── CO2 / PM2.5 传感器
├── 压差传感器 / 流量计 / 风速传感器
├── DDC 控制器
└── 变频器（VFD）
```

远程操作：监控传感器数据、温度设定、风机启停、阀门开度、模式切换、时间表排程、最优启停、能效诊断

### B. 照明系统（Lighting）

```
├── 灯具（LED/荧光灯/应急灯）
├── 照明控制器 / 调光器
├── 人体感应器 / 光照度传感器
├── 场景面板
├── 应急照明 / 疏散指示
└── 室外景观/立面/道路照明
```

远程操作：开关、调光、场景切换、光感/人感联动、定时排程、能耗统计

### C. 电气系统（Electrical）

```
供配电
├── 变压器 / 高低压配电柜
├── 电力电缆 / 母线
└── 电能表 / 多功能表

应急电源
├── 柴油发电机
├── UPS / EPS
└── 蓄电池组

新能源
├── 光伏（PV）
├── 储能（BESS）
└── 充电桩（EV Charger）
```

远程操作：电参数监控、回路能耗、负载分析、峰谷策略、发电机状态、充电桩管理

### D. 给排水（Plumbing & Water）

```
├── 市政进水 / 水箱水池 / 增压泵组 / 水表
├── 热水系统（锅炉/热泵/太阳能/循环泵）
├── 排水系统（污水泵/集水坑/隔油池）
├── 雨水回收 / 中水回用
└── 水质监测（pH/浊度/余氯）
```

远程操作：液位监控、水泵状态、水压水温、漏水告警、水质数据

### E. 电梯与扶梯（Vertical Transportation）

```
├── 客梯 / 货梯 / 消防电梯
├── 自动扶梯 / 自动步道
└── 电梯群控系统
```

远程操作：运行状态、楼层位置、故障告警、困人检测（控制权受法规限制）

### F. 消防系统（Fire Protection）

```
自动报警：烟感/温感/手报/声光/报警主机
灭火：喷淋/气体灭火/消火栓/消防泵/消防水箱
防排烟：排烟风机/加压送风/排烟阀/防火阀/挡烟垂壁
其他：防火卷帘/应急广播/消防电话
```

远程操作：**仅限监控**（法规限制）— 报警事件、设备状态、巡检记录、误报统计

### G. 安防系统（Security）

```
视频监控：IP 摄像头 / NVR / AI 视频分析
门禁：读卡器/电锁/闸机/人脸识别/访客管理
入侵报警：红外对射/玻璃破碎/振动传感器
巡更：巡更点/路线/记录
```

远程操作：视频查看回放、远程开门、权限下发、布撤防、巡更管理

### H. 停车场（Parking）

```
├── 车牌识别（LPR）/ 道闸
├── 车位引导 / 车位传感器
├── 收费系统 / 月卡管理
└── 反向寻车
```

远程操作：车位占用率、远程抬杆、费率策略、黑白名单

### I. ICT 网络与机房（IT Infrastructure）

```
├── 结构化布线 / 网络交换机 / 路由器
├── 无线 AP / Wi-Fi 管理
├── 机房环境监控（温湿度/UPS/漏水/烟感）
├── 公共广播 / 背景音乐（PA/BGM）
└── 数字标牌（Digital Signage）
```

远程操作：网络监控、AP 管理、机房环境、广播控制、标牌内容推送

### J. 能源管理（Energy Management）

```
├── 分项计量（照明/空调/动力/特殊/燃气/水）
├── 能耗分析 / 能效对标
├── 碳排放核算
├── 需求响应（Demand Response）
└── 能源审计
```

### K. 环境与舒适度（IEQ）

```
├── 温度 / 湿度 / CO2 / PM2.5 / PM10
├── TVOC / 甲醛 / 氡
├── 噪音 / 光照度
└── 气流速度 / 热辐射
```

### L. 建筑外围护（Building Envelope）

```
├── 幕墙系统（检测/维护）
├── 电动遮阳（百叶/卷帘/天棚帘）
├── 电动开窗器（自然通风）
├── 屋顶绿化灌溉系统
└── 外墙保温监测
```

远程操作：遮阳角度、开窗控制、灌溉定时

### M. 垂直物流与废弃物（Material Handling）

```
├── 气动垃圾收集系统（Pneumatic Waste）
├── 物流传输系统（如医院气动物流）
├── 垃圾分类压缩站
└── AGV 自动导引车（仓储/工厂）
```

## 三、各楼宇类型的独有运营场景

> 以下按楼宇类型逐一展开，只列出该类型**独有的或特别重要的**运营场景，通用系统不再重复。

---

### 3.1 酒店（Hotel）

#### 独有系统与设备

```
客房管理系统（Guest Room Management System — GRMS）
├── 客房控制器（Room Controller）
│   ├── 空调控制（温度设定、风速、模式）
│   ├── 照明场景（欢迎/阅读/睡眠/全开/全关）
│   ├── 窗帘/遮阳帘电动控制
│   ├── 取电开关（Key Card Switch）
│   ├── 勿扰/请清理（DND/MUR 指示）
│   └── 迷你吧（Minibar 门感应）
├── 客房状态面板
│   ├── 入住/退房/空房/维修
│   └── 客房温度预设（入住前提前开启）
└── 客房能耗计量（按房间）

热水系统（大规模集中供热水）
├── 中央热水锅炉组 / 热泵热水器
├── 太阳能集热器
├── 储热水箱（几十吨级）
├── 热水循环泵（保证即开即热）
└── 恒温控制 / 防军团菌消毒（定期 60°C+ 冲洗）

洗衣房系统
├── 工业洗衣机 / 烘干机
├── 蒸汽发生器
├── 水处理（软化水）
├── 排水排气
└── 能耗/水耗计量

厨房与餐饮
├── 商业厨房排油烟系统
├── 冷库/冰柜温度监控
├── 燃气泄漏检测
├── 灶台灭火系统
├── 餐厅照明/空调场景
└── 食品安全温度记录（HACCP）

泳池与水疗（Spa/Pool）
├── 泳池水温控制
├── 水质监测（pH/余氯/ORP/浊度）
├── 循环过滤泵
├── 加热系统（热泵/换热器）
├── 桑拿/蒸汽房温控
└── 更衣室通风除湿

会议/宴会设施
├── 大宴会厅分隔系统（活动隔断墙）
├── 专业灯光/音响/投影
├── 同声传译系统
└── 按预约联动空调/照明

停车与交通
├── 代客泊车管理
├── VIP 车辆识别
└── 接驳车调度
```

#### 酒店独有运营场景

| 场景 | 描述 | 涉及系统 |
|------|------|---------|
| **客房入住联动** | PMS 通知入住 → 提前开空调到设定温度 → 欢迎灯光场景 → 电子门牌显示客人姓名 | GRMS + PMS + HVAC + 照明 + 门禁 |
| **客房退房联动** | PMS 通知退房 → 全关空调/灯光/窗帘 → 状态切换为"待清洁" | GRMS + PMS |
| **客房节能** | 取电开关拔出 → 延时 15 分钟关空调/灯光（保留冰箱/充电） → 回温/回冷设定 | GRMS + 能源 |
| **VIP 预设** | 高级客人入住前按偏好预设温度、灯光、窗帘位置、迷你吧补货 | GRMS + CRM |
| **热水保障** | 保证任意时段、任意房间打开龙头 5 秒内出热水，回水温度监控 | 热水系统 |
| **军团菌防控** | 定期检测水温，低于 50°C 告警，定期 60°C+ 冲洗全管路 | 热水 + 水质 |
| **泳池水质自动投药** | pH/余氯偏离目标 → 自动投加药剂 → 记录合规 | 泳池系统 |
| **大型活动模式** | 宴会厅 500 人活动 → 加大新风 → 调低温度 → 灯光宴会场景 → 电梯定向服务 | HVAC + 照明 + 电梯 |
| **深夜模式** | 23:00 后公区照明降 50% → 空调降频 → 大堂保留基础照明 → 安防加强 | 照明 + HVAC + 安防 |
| **紧急疏散** | 火警 → 客房灯全开 + 广播疏散 → 电梯迫降 → 排烟启动 → 门禁全开 | 消防 + GRMS + 电梯 + 门禁 |

#### 酒店特有集成

| 对接系统 | 用途 |
|---------|------|
| PMS（Property Management System） | 客房状态、入住退房联动 |
| POS（收银系统） | 餐饮/水疗消费 |
| 客房 IPTV / 客控面板 | 客人交互界面 |
| OTA 平台（携程/Booking） | 房态同步 |
| CRM | VIP 偏好记录 |
| 门锁系统（Assa Abloy/Onity） | 电子房卡发卡/回收 |

### 3.2 医院（Hospital）

#### 独有系统与设备

```
医用气体系统（Medical Gas）
├── 中心供氧（Oxygen — O2）
├── 负压吸引（Vacuum/Suction）
├── 压缩空气（Medical Air）
├── 笑气/氧化亚氮（N2O）
├── 二氧化碳（CO2，腹腔镜用）
├── 氮气（N2，冷冻治疗）
├── 汇流排/液氧罐
├── 管道压力监测（每个区域）
├── 气体报警面板（每个护士站）
└── 手术室专用气体面板

洁净环境控制
├── 手术室洁净度分级
│   ├── I 级（百级层流）— 关节置换、器官移植
│   ├── II 级（千级）— 一般外科
│   ├── III 级（万级）— 门诊手术
│   └── IV 级（三十万级）— 换药室
├── ICU/NICU/CCU 洁净控制
├── 正压/负压控制
│   ├── 手术室：正压（防外部污染进入）
│   ├── 传染病房：负压（防病原外泄）
│   └── 压差梯度：走廊 → 缓冲 → 手术室
├── HEPA 高效过滤（H13/H14）
├── 换气次数监控（手术室 20-25 次/h）
├── 温湿度精密控制（22-25°C，40-60%RH）
└── 粒子计数器（实时洁净度监测）

放射防护
├── 放射科/核医学科屏蔽
├── 射线监测（辐射剂量）
├── 防护门联锁（设备运行时门不能开）
├── 排气系统（放射性气体）
└── 废水衰变池

医用冷链
├── 血库冰箱（4°C ± 2°C，断电告警）
├── 药品冷藏（2-8°C）
├── 超低温冰箱（-80°C，样本/疫苗）
├── 手术室低温冷冻设备
└── 太平间冷藏

气动物流传输（Pneumatic Tube System）
├── 发送站 / 接收站
├── 传输管道网络
├── 控制器 / 分拣转换器
└── 紧急优先传输（手术室/急诊）

净化水系统
├── 纯化水制备（反渗透 RO）
├── 血透用水（超纯水）
├── 供应室用水（灭菌）
├── 水质在线监测（电导率/TOC/细菌）
└── 分配管路循环

医疗废物管理
├── 感染性废物（黄色袋）
├── 损伤性废物（利器盒）
├── 化学性废物
├── 放射性废物
├── 暂存间温控（防腐败）
├── 称重/分类记录
└── 转运追踪

手术室专属
├── 无影灯 / 吊塔
├── 手术室内对讲/示教
├── 手术排程联动（空调提前开启/术后快速换气）
├── 器械追踪（清洗-灭菌-使用-回收）
└── 麻醉废气排放（WAGD）
```

#### 医院独有运营场景

| 场景 | 描述 | 涉及系统 |
|------|------|---------|
| **手术室准备联动** | 手术排程确认 → 提前 1h 开启层流 → 温度调至 22°C → 湿度 50% → 医气检查 → 洁净度达标确认 | HVAC + 洁净 + 医气 |
| **传染病隔离** | 确诊传染病 → 病房切负压模式 → 压差监控 → 门禁限制 → 废弃物专用通道 | HVAC + 门禁 + 废物 |
| **血库温度异常** | 血库冰箱温度 >6°C → 紧急告警 → 通知值班 → 备用冰箱启动 → 记录断链时间 | 冷链 + 告警 |
| **供氧压力低** | 汇流排/液氧罐压力低于阈值 → 护士站报警 → 通知设备科 → 备用切换 | 医气 |
| **停电应急** | 市电中断 → 发电机 10 秒内启动 → 保障手术室/ICU/急诊/血库/医气 → 非关键区域切断 | 电气 + 所有关键系统 |
| **消毒净化周期** | 手术后 → 快速换气（15-20 分钟全排全新风）→ 紫外消毒 → 洁净度自检 → 下台手术可用 | HVAC + 洁净 |
| **放射科联锁** | 放射设备启动 → 防护门关闭确认 → 警示灯亮 → 通风启动 → 辐射监测 | 放射 + 门禁 + 通风 |
| **新生儿温控** | NICU 恒温 25-28°C、湿度 50-65%，每个暖箱独立监控，告警直达护士站 | HVAC + 监控 |
| **医废追踪** | 废物产生→称重→暂存→转运→处置，全程追踪，合规记录 | 废物管理 |
| **疫情大规模筛查** | 临时设置发热门诊 → 独立通风（不回风）→ 负压帐篷 → 独立出入口 | HVAC + 门禁 + 临时设施 |

#### 医院特有集成

| 对接系统 | 用途 |
|---------|------|
| HIS（医院信息系统） | 手术排程、病房管理 |
| PACS（影像系统） | 机房环境联动 |
| 护理呼叫系统 | 护士站告警 |
| 手术示教系统 | 视频/音频 |
| 药房自动化 | 气动物流、自动药柜 |
| 消毒供应追溯 | 器械全生命周期 |

#### 医院特有合规要求

- 手术室洁净度必须有连续记录，可追溯
- 医气系统压力记录保存 ≥3 年
- 冷链温度记录保存 ≥5 年
- 医疗废物转运联单
- 放射防护年度检测报告
- 院感数据上报（传染病房空气培养）

### 3.3 商场/购物中心（Shopping Mall）

#### 独有系统与设备

```
租户管理
├── 租户独立计量（电/水/燃气/空调）
├── 租户用电限容（超负荷自动报警）
├── 公摊费用分摊算法
├── 租户装修管理（施工期间临时配电/通风）
└── 租户空调计费（按冷量/按面积/按时间）

客流分析
├── 客流计数器（入口/楼层/区域）
├── Wi-Fi 探针 / 蓝牙信标
├── 视频客流分析（AI）
├── 热力图（人流密度）
├── 驻留时间分析
└── 动线分析（顾客路径）

中庭/大空间环境
├── 大空间空调（分层送风/地板送风/座椅送风）
├── 中庭自然通风（电动天窗/排烟窗兼用）
```

| **机柜上架** | 新设备安装 → 确认电力/冷量/空间余量 → 分配 PDU 端口 → 开机 → 监控功耗变化 | 容量 + 电气 |
| **自然冷却切换** | 室外温度下降 → 自动开启水侧自然冷却 → 关闭/减载冷机 → 持续监控机房温度 | HVAC + 气象 |
| **发电机联动** | 市电中断 → 10s 内发电机启动 → ATS 切换 → 市电恢复 → 延时回切 → 记录事件 | 电气 |

#### 数据中心特有指标

- **PUE**（Power Usage Effectiveness）：总能耗/IT 能耗，目标 <1.3
- **WUE**（Water Usage Effectiveness）：用水效率
- **可用性**：Tier I (99.671%) → Tier IV (99.995%)
- **温度范围**：ASHRAE A1 推荐 18-27°C

## 六、通信协议与集成

### 6.1 楼宇侧协议

| 协议 | 场景 | 特点 |
|------|------|------|
| **BACnet/IP** | 楼宇自控主流 | 标准化强，对象模型完整 |
| **BACnet MS/TP** | DDC 到传感器 | RS-485 总线 |
| **Modbus TCP/RTU** | 电力仪表、传感器 | 简单通用，无语义 |
| **OPC-UA** | 工业自动化、新型 BMS | 安全性好，信息模型丰富 |
| **KNX/EIB** | 照明、窗帘、面板 | 欧洲标准，分布式 |
| **DALI** | 照明专用 | 灯具级别控制 |
| **LonWorks** | 老旧楼控系统 | 逐渐被 BACnet 替代 |
| **MQTT** | IoT 传感器、边缘设备 | 轻量、适合大量点位 |
| **HTTP/REST** | 新型设备/子系统 API | 易集成 |
| **ONVIF** | 视频监控 | 摄像头互操作标准 |
| **OSDP** | 门禁 | 新一代门禁协议（替代 Wiegand） |
| **Zigbee/Z-Wave/蓝牙** | 智能家居/传感器 | 低功耗无线 |
| **LoRa/NB-IoT** | 大范围低频传感 | 抄表、环境监测 |
| **私有协议** | 品牌设备 | 电梯（OTIS/迅达）、消防（霍尼韦尔/海湾） |

## 九、关键设计决策（待确定）

1. **数据存储架构？**
   - 边缘（容器内）：实时数据 + 短期历史（7-30 天）
   - 云端（中心）：长期历史 + 分析 + 报表
   - 还是纯边缘（每栋楼独立，适合安全敏感客户）？

2. **实时通信方案？**
   - 当前 SSH 隧道 — 适合开发调试
   - 生产建议：MQTT broker + WebSocket（大量点位推送）
   - 安全敏感场景（医院/警局）：VPN + 加密

3. **点位数据模型？**
   - 自定义 vs 行业标准（Project Haystack / Brick Schema）
   - 建议至少参考 Haystack 标签体系

4. **多租户？**
   - SaaS 多租户（一套系统管多个客户的多栋楼）
   - 私有化部署（每个客户独立）
   - 混合（核心 SaaS + 边缘私有）

5. **安全与合规？**
   - 控制操作审批流程（谁能控制什么）
   - 消防等系统的只读限制
   - 不同楼宇类型的合规记录要求差异
   - 数据加密（传输 + 存储）
   - 等保/GDPR 等

6. **楼宇模板机制？**
   - 预置模板（酒店/医院/商场...）包含什么？
   - 子系统列表 + 标准点位 + 默认告警规则 + 标准排程 + 合规要求
   - 用户可基于模板自定义
