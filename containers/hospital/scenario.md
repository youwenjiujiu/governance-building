# 医院行业场景定义

> **Container**: `hospital`
> **定位**: 楼宇远程运营平台医院行业 Container，覆盖医用气体、洁净室、冷链、放射防护、净化水等医院专属系统。
> **适用场景**: 综合医院（二甲及以上）的关键基础设施远程运维，保障医疗安全与合规。

## 2. 标准告警规则

### 2.1 医用气体告警

| 告警名称 | 条件 | 严重度 | 处理建议 |
|---------|------|--------|---------|
| O2管网低压 | O2_Floor_P < 300kPa 持续2min | 紧急 | 检查液氧站/汇流排，通知手术室，启动备用 |
| O2管网高压 | O2_Floor_P > 450kPa | 严重 | 检查减压阀，关闭上游阀门 |
| 液氧液位低 | LOX_Level < 25% | 警告 | 联系供气商补液 |
| 液氧液位极低 | LOX_Level < 15% | 紧急 | 立即补液，切换汇流排备用 |
| 真空度不足 | VAC_Pipe_P > 60kPa(abs) 持续3min | 紧急 | 检查真空泵，手术室可能受影响 |
| 真空泵全停 | 所有VAC_Pump_St=OFF/FAULT | 紧急 | 最高级别应急，通知手术室暂停 |
| 压缩空气低压 | MA_Pipe_P < 350kPa 持续2min | 紧急 | 检查空压机，影响呼吸机供气 |
| 压缩空气露点高 | MA_Dewpoint > -20°C | 警告 | 检查干燥器，水分超标 |
| 压缩空气含油超标 | MA_Oil > 0.1mg/m³ | 严重 | 检查空压机，医用空气不合格 |
| N2O低压 | N2O_Pipe_P < 300kPa 持续2min | 严重 | 更换气瓶，通知麻醉科 |
| 气体报警面板故障 | 面板通讯丢失>5min | 严重 | 现场检查面板，人工监测 |

### 2.2 洁净室告警

## 4. 运营场景剧本

### 场景1：手术室启用（首台手术）

**触发条件**: 工作日06:00，手术排程系统确认当日有手术

**步骤**:
1. 06:00 — 净化AHU全速启动，新风阀/回风阀开启
2. 06:05 — 检查送风量达标，确认HEPA差压正常
3. 06:10 — 温湿度调节至设定值（22°C / 50%RH）
4. 06:15 — 确认室内-走廊压差 ≥+5Pa（百级≥+8Pa）
5. 06:30 — 粒子计数器自检，确认达到对应洁净等级
6. 06:45 — 医用气体压力确认（O2/真空/压缩空气/N2O）
7. 07:00 — 系统自检通过，标记手术室"可用"，通知手术室护士
8. 自检不通过 → 生成紧急告警，标记手术室"不可用"，通知设备科

### 场景2：手术室换台

**触发条件**: 一台手术结束，下一台即将开始

**步骤**:
1. T+0min — 手术结束标记，启动自净程序
2. T+1min — AHU维持全速，不降频/不停机
3. T+5min — 清洁人员进入清洁消毒
4. T+15min — 自净完成，粒子计数器验证洁净度
5. T+15~20min — 温湿度/压差复核
6. 验证通过 → 标记手术室"可用"
7. 验证不通过 → 延长自净时间至30min，重新检测

### 场景3：传染病隔离手术/病房

**触发条件**: 院感科确认传染病患者需手术或隔离

**步骤**:
1. 将目标手术室/病房切换为负压模式
2. 关闭回风阀，开启全排风阀
3. 确认室内-走廊压差 ≤-5Pa（负压建立）
4. 启动AGSS或独立排风至室外高空排放
5. 联动门禁：设置二次缓冲间正压
6. 粒子/气溶胶监测加密至每5min
7. 手术/治疗结束后：维持负压30min终末消毒
8. 终末消毒后：恢复正常正压模式，粒子验证后释放

### 场景4：供氧低压应急

**触发条件**: O2_Floor_P < 300kPa 或 LOX_Level < 15%

**步骤**:
1. T+0s — 气体报警面板声光报警，系统推送紧急告警
2. T+30s — 确认液氧站/汇流排状态，自动切换至备用气源
3. T+1min — 通知手术室、ICU、急诊科降低非紧急用氧量
4. T+2min — 设备科人员赶赴液氧站/汇流排现场
5. T+5min — 评估故障原因（泄漏/减压阀故障/供气中断）
6. 如为液氧耗尽 → 紧急联系供气商送氧
7. 如为管路泄漏 → 关闭泄漏段区域阀箱，隔离故障
8. 恢复供气后 → 逐层确认压力恢复，解除报警

### 场景5：血库温度异常

**触发条件**: BB_Temp > 6°C 持续5min 或 BB_Temp < 1°C

**步骤**:
1. T+0s — 系统推送紧急告警至血库值班+设备科
2. T+1min — 血库人员现场确认温度计显示
3. T+2min — 检查制冷系统状态（压缩机/冷凝器/门是否关好）
4. T+5min — 如无法恢复 → 启动血液转移预案
5. 血液转移至备用冰箱或其他科室冰箱
6. 记录温度偏离时间段及受影响血液批号
7. 通知输血科评估受影响血液是否可用
8. 修复后 → 温度稳定2h确认，恢复使用

### 场景6：停电应急

**触发条件**: 市电中断

**步骤**:
1. T+0s — UPS自动接管生命支持设备（呼吸机/监护仪/血库）
2. T+10s — 柴油发电机自动启动（ATS切换）
3. T+15s — 确认发电机带载：手术室AHU/ICU/血库/急诊照明
4. T+30s — 非关键负载断开（门诊空调/普通照明/电梯部分）
5. T+2min — 系统确认所有关键系统供电正常
6. T+5min — 通知全院停电状态及预计恢复时间
7. 发电机运行期间 → 监控柴油液位、发电机温度/负载
8. 市电恢复 → ATS切回市电，发电机冷却后停机
9. 事后 → 记录停电时间/时长/影响范围，生成报告

### 场景7：放射联锁故障

**触发条件**: 放射设备准备出束时防护门联锁异常

**步骤**:
1. T+0s — 联锁系统阻止出束，生成紧急告警
2. T+30s — 放射科技师现场确认防护门关闭状态
3. T+1min — 检查联锁传感器/控制器/线路
4. 如为门限位开关故障 → 现场更换/调整后复位测试
5. 如为控制器故障 → 通知设备科检修，暂停该机房使用
6. 复位后 → 完整联锁测试（开门→锁门→出束→开门断束）
7. 测试通过 → 恢复使用，记录故障原因
8. 测试不通过 → 停用该机房，患者转至其他机房

### 场景8：院感巡查模式

**触发条件**: 院感科启动定期/临时巡查

**步骤**:
1. 系统一键生成全院洁净区域状态报告
2. 报告内容：各手术室/ICU/产房温湿度/压差/粒子/HEPA差压
3. 高亮超标或临近阈值的区域
4. 院感人员现场采样比对（沉降菌/浮游菌）
5. 发现异常 → 生成整改工单，指定责任人和完成时限
6. 整改完成 → 复测确认，关闭工单
7. 巡查结果归档，作为等级评审材料

### 场景9：NICU温湿度异常

**触发条件**: NICU区域温度 > 27°C 或 < 23°C 或湿度 > 65%

**步骤**:
1. T+0s — 系统推送告警至新生儿科+设备科
2. T+1min — 护士现场确认暖箱/辐射台设定无误
3. T+2min — 检查净化AHU运行状态、冷/热水阀开度
4. T+5min — 如AHU故障 → 启动备用AHU或便携式空调应急
5. T+10min — 确认NICU温度恢复趋势
6. 如持续异常 → 与新生儿科协调是否转移患儿
7. 修复后 → 持续监测2h，确认稳定

### 场景10：疫情筛查模式

**触发条件**: 卫健委通知/院内确认传染病疫情

**步骤**:
1. 启动发热门诊独立空调（全新风+负压+高效过滤）
2. 隔离病房切换负压模式（参考场景3）
3. 门诊/急诊入口通风加强（开启全新风模式）
4. 洁净区域换气次数提升至标准上限
5. 全院新风量增至最大（降低气溶胶浓度）
6. 加密冷链监测频率（可能涉及核酸样本冷藏）
7. 医用气体用量监测（供氧量可能剧增）
8. 每日生成通风/消毒/压差状态日报给院感科
9. 疫情解除 → 全院终末消毒 → 恢复正常运行模式

| 签名确认 | 关键操作需电子签名（操作人+审核人） | GMP / 等级评审 |
| 导出格式 | 支持PDF/Excel导出，含时间戳+数字签名 | 监管检查要求 |

## 7. 与HIS集成

### 7.1 集成架构

```
HIS(医院信息系统)
 ├── 手术排程系统 ──→ 手术室净化AHU启停联动
 ├── ADT(入院/出院/转科) ──→ 病房空调/负压模式切换
 ├── 院感监测系统 ──→ 隔离病房负压控制触发
 ├── 血库管理系统 ──→ 冷链报警关联血液批号追溯
 ├── 药品管理系统 ──→ 冷链温度异常关联药品批号
 ├── 输血科系统 ──→ 血库温度偏离时效评估
 ├── 放射科RIS ──→ 防护门联锁/辐射监测联动
 └── PACS影像系统 ──→ 机房温湿度监控
```

### 7.2 关键接口

| 接口名称 | 方向 | 协议 | 频率 | 数据内容 |
|---------|------|------|------|---------|
| 手术排程同步 | HIS→BMS | HL7/REST | 实时 | 手术室号/开始时间/结束时间/手术类型 |
| 患者隔离标记 | HIS→BMS | HL7/REST | 实时 | 患者ID/病房号/隔离类型(空气/飞沫/接触) |
| 冷链告警推送 | BMS→HIS | REST/MQ | 实时 | 设备ID/温度/告警级别/受影响物品 |
| 环境数据回传 | BMS→HIS | REST | 15min | 手术室/ICU温湿度压差粒子快照 |
| 血液温度追溯 | BMS→血库 | REST | 按需 | 时间段+温度曲线+偏离记录 |
| 辐射剂量同步 | BMS→RIS | REST | 每次检查 | 房间剂量率/累积剂量/联锁状态 |

### 7.3 数据安全要求

- 所有接口需HTTPS加密传输
- 患者信息脱敏（仅传递患者ID，不传姓名/身份证）
- 接口访问通过OAuth2.0认证
- 操作日志保留≥5年（等保三级要求）
- 符合《个人信息保护法》及《医疗数据安全管理规范》

### 8.4 缺口分析

| 缺口项 | 说明 | 建议 |
|--------|------|------|
| 患者关联 | base层无患者实体，医院需关联患者ID用于追溯 | 通过HIS接口桥接，BMS仅存患者ID不存敏感信息 |
| 医疗设备注册管理 | base层Equipment无注册证号/分类字段 | 通过`metadata`扩展或新增字段 |
| 双人操作审计 | base层AuditLog为单操作人，关键操作需操作人+审核人 | 扩展AuditLog增加`reviewer_id`和`reviewer_at` |
| 法规版本管理 | ComplianceRule需关联具体法规版本号及生效日期 | 扩展增加`regulation_version`和`effective_date` |
| 紧急广播联动 | base层无广播/PA系统集成 | 新增Notification渠道类型`PA_BROADCAST` |
| 负压/正压模式切换逻辑 | base层Schedule无正负压切换概念 | 通过Scenario实体的`mode`字段实现 |
