# 写字楼 HVAC 行业场景定义

> **Container**: `office-hvac`
> **定位**: 楼宇远程运营平台第一个行业 Container，专注于写字楼的 HVAC（暖通空调）运营管理。
> **适用场景**: 甲级/乙级写字楼的中央空调系统远程运维，涵盖冷站、输配、末端、控制全链路。

---

├── 变频驱动 (VFD)
│   ├── 冷冻水泵 VFD × N

## 2. 标准告警规则

### 2.1 冷水机组告警

| 告警名称 | 条件 | 严重度 | 处理建议 |
|---------|------|--------|---------|
| 冷冻水供水温度偏高 | CHWS_Temp > SP + 2°C 持续 10min | 警告 | 检查冷机负载、冷冻水流量、冷凝器清洁度 |
| 冷冻水供水温度过高 | CHWS_Temp > SP + 5°C 持续 5min | 严重 | 检查冷机运行台数是否足够，考虑加机 |
| 冷却水温度过高 | CWR_Temp > 37°C | 警告 | 检查冷却塔运行状态、风机频率 |
| 冷机 COP 过低 | COP < 3.5 持续 30min | 警告 | 检查冷凝器/蒸发器结垢、制冷剂量 |
| 压缩机过载 | Comp_Load > 95% 持续 15min | 警告 | 考虑加机分担负载 |
| 冷机故障 | CH_Status = FAULT | 紧急 | 立即切换备用机，派人检查故障码 |
| 防冻保护触发 | CH_Freeze = ALARM | 紧急 | 检查冷冻水流量、水温，防止盘管冻裂 |
| 油温异常 | Oil_Temp > 65°C 或 < 25°C | 警告 | 检查油加热器/油冷却器 |
| 高低压差不足 | (Cond_Press - Evap_Press) < 阈值 | 警告 | 可能制冷剂泄漏 |
| 运行时间不均衡 | 多台冷机运行时间差 > 20% | 信息 | 调整启停轮换策略 |

### 2.2 冷却塔告警

| 新风系统故障 | 多区域 CO2 同时升高 + MAU 停机/故障 | 严重 | 新风机组故障导致全楼新风不足 |
| 冷却水系统异常 | 冷机 COP 下降 + 冷却水温度偏高 + 冷却塔正常 | 警告 | 可能冷凝器结垢需清洗 |
| 末端大面积离线 | 同一 DDC 控制器下多台 FCU/VAV 离线 | 严重 | DDC 控制器或通讯网络故障 |
| 租户区域异常 | 某租户区域多台 FCU 异常 + 相邻租户正常 | 警告 | 该租户管路阀门/水力平衡问题 |
| 全楼制冷失效 | 所有冷机故障/停机 + 室内温度快速上升 | 紧急 | 启动应急预案，通知物业/租户 |

### 3.6 季节切换

#### 制冷→过渡季切换（秋季）

```
触发条件：连续 5 天日均室外温度 < 22°C 且 最高温度 < 26°C

切换步骤：
1. 冷站停机（保留 1 台冷机备用）
2. AHU 切换至全新风模式（经济器模式）
3. FCU 水阀关闭，风机可继续运行（通风）
4. MAU 停止冷水供应，仅送新风
5. 监控室内温度，如超温则恢复部分冷站
```

#### 过渡季→制热切换（冬季）

```
触发条件：连续 3 天日均室外温度 < 10°C 且 最低温度 < 5°C

切换步骤：
1. 启动热源（锅炉/热泵/市政供热二级泵站）
2. AHU 切换至冬季模式（新风预热、最小新风量）
3. FCU 热水阀启用
4. 防冻保护全面启用
5. 冷却塔排空/防冻处理
6. 冷水机组保养维护
```

#### 制热→过渡季切换（春季）

```
触发条件：连续 5 天日均室外温度 > 12°C 且 最低温度 > 5°C

切换步骤：
1. 热源停机
2. AHU 切换至全新风模式
3. FCU 热水阀关闭
4. 冷站准备（冷机试运行、冷却塔补水清洗）
```

## 4. 运营场景剧本（Playbook）

### 场景1：正常工作日运营

**触发条件**: 工作日 + 非极端天气 + 所有设备正常

**涉及设备**: 冷站全套 + AHU/MAU + FCU/VAV + BMS

**操作时间线**:

```
06:00  BMS 自动检查：
       ├── 读取室外温湿度 → 决定当天运行策略
       ├── 检查设备状态（是否有隔夜故障）
       └── 根据天气预报预判峰值负荷

06:30  预冷启动：
       ├── 启动冷冻水泵（低频 25Hz）
       ├── 启动 1 台冷机（冷冻水供水设定 7°C）
       ├── 启动冷却水泵 + 冷却塔
       ├── AHU 送风机低速启动（30Hz）
       └── 大堂/电梯厅 FCU 开启

07:30  检查预冷效果：
       ├── 大堂温度应降至 25°C 以下
       ├── 如不达标 → 加开第 2 台冷机
       └── MAU 新风机组启动

08:00  全面运行：
       ├── 冷站根据负荷自动加机（通常 2~3 台）
       ├── AHU 全速运行
       ├── 所有楼层 FCU/VAV 按排程开启
       ├── 冷冻水泵频率根据管网压差自动调节
       └── 运维人员巡查大屏：确认无异常告警

10:00  上午峰值负荷：
       ├── 关注冷站负载率（通常 70~90%）
       ├── 关注 AHU 送风温度达标
       └── 处理零星租户投诉（温度偏差调整）

12:00  午休节能：
       ├── 系统自动减 1 台冷机（如负荷允许）
       ├── AHU 降频运行（40Hz→35Hz）
       └── 部分公区灯光联动减亮

13:00  午后恢复：
       ├── 负荷恢复，冷站加机
       └── AHU 恢复全速

14:00~16:00  全天负荷峰值：
       ├── 西向楼层负荷增大（西晒）
       ├── 冷站满负荷运行
       ├── 关注冷冻水供水温度稳定性
       └── 必要时降低冷冻水供水温度设定（7°C→6.5°C）

17:30  下班过渡：
       ├── 冷站根据负荷自动减机
       ├── 无人楼层 FCU 自动关闭（占用传感器/排程）
       ├── AHU 降频
       └── 系统提示：有无加班延时申请？

19:00  加班时段：
       ├── 仅加班申请区域保持供冷
       ├── 冷站维持最少运行台数
       └── 其余区域设备停止

22:00  全面停机：
       ├── 冷站停机
       ├── AHU/MAU 停机
       ├── FCU/VAV 全部关闭
       ├── 冷冻水泵/冷却水泵停止
       └── 系统进入夜间监控模式（仅告警）
```

**判断逻辑**:
- 冷机加减机依据：冷冻水回水温度 vs 设定值、冷机负载率
- AHU 变频依据：送风静压反馈 vs 设定值
- 冷冻水泵变频依据：最不利环路压差 vs 设定值
- 冷却塔风机变频依据：冷却水回水温度 vs 设定值

**预期结果**:
- 办公区域 23~26°C（夏季）
- CO2 < 1000ppm
- 相对湿度 40~65%
- 冷站效率 COP > 4.5
- 零未处理紧急告警

### 场景2：加班延时空调申请

**触发条件**: 租户通过 APP/电话申请延时空调

**涉及设备**: 申请区域的 FCU/VAV + 对应 AHU（如需） + 冷站（如需）

**操作步骤**:

```
1. 接收申请
   ├── 租户 APP 提交：公司名、楼层、区域、时间段（如 18:00~21:00）
   └── 系统自动识别该区域涉及的设备

2. 费用估算
   ├── 查询费率表（工作日延时 vs 周末 vs 节假日）
   ├── 计算涉及设备运行费用
   ├── 判断冷站是否已在运行（共享冷站无需额外冷站费用）
   └── 向租户展示预估费用

3. 审批
   ├── 自动审批条件：金额 < 500元 且 工作日延时
   └── 超限则推送至物业主管审批

4. 设备联动
   ├── T-10min：预启动冷站（如当前已停机）
   ├── T-0min：开启指定区域 FCU/VAV + 对应 AHU + MAU
   ├── 冷冻水泵频率自动调节
   └── 运行期间正常告警监控

5. 自动关闭
   ├── T+结束时间：设备自动停止
   ├── 延时提醒：结束前 15min 推送通知"空调即将关闭"
   └── 如需延长：租户可 APP 续约

6. 计费结算
   ├── 记录实际开启/关闭时间（取整至 30min）
   ├── 按费率计算实际费用
   └── 纳入月度账单
```

**判断逻辑**:
- 如果当前已有其他租户在加班，冷站已运行，新申请仅需开启末端
- 如果是最后一个加班租户结束，需关闭冷站
- 凌晨申请需人工审批（安保协调）

**预期结果**:
- 申请区域温度恢复正常 < 15min
- 费用准确计入租户账单
- 无人为干预的设备联动

### 场景3：会议室联动

**触发条件**: 会议室预约系统确认会议即将开始 / 人体传感器检测到入室

**涉及设备**: 会议室 FCU/VAV + 新风 + 照明

**操作步骤**:

```
1. 会议前预冷（会议开始前 15min）
   ├── 会议室 FCU 切换至高速
   ├── VAV 风阀开至最大
   ├── 温度设定值下调 1°C（预冷补偿）
   └── 新风量增至最大（预通风）

2. 会议进行中
   ├── 根据人数动态调节（CO2 反馈）
   │   ├── CO2 < 800ppm：新风维持
   │   ├── CO2 800~1000ppm：新风增大
   │   └── CO2 > 1000ppm：新风最大 + 告警
   ├── 温度 PID 正常控制
   └── 人数超设计容量 → 提醒物业

3. 会议结束
   ├── 预约系统标记结束 或 人体传感器 10min 无人
   ├── FCU 切换至低速或关闭
   ├── VAV 回到最小风量
   └── 温度设定恢复默认
```

**判断逻辑**:
- 大会议室（>20人）需提前 30min 预冷
- 连续会议（间隔 <15min）不关闭设备
- VIP 会议室独立温度策略

**预期结果**:
- 会议开始时室温已达标
- 会议全程 CO2 < 1000ppm
- 会议结束后 10min 内设备回到节能模式

### 场景4：夏季极端高温日

**触发条件**: 室外温度 > 38°C 或 气象预警发布高温橙色/红色预警

**涉及设备**: 冷站全套 + 全部末端 + BMS

**操作步骤**:

```
1. 提前准备（前一天晚间/当日凌晨）
   ├── 检查所有冷机状态（确保 100% 可用）
   ├── 冷却塔提前清洗/检查
   ├── 确认备用冷冻水泵/冷却水泵可用
   ├── 蓄冷罐充满（如有蓄冷系统）
   └── 通知运维团队进入高温保障模式

2. 提前预冷（比正常提早 1h）
   ├── 05:30 启动冷站（比正常早 1h）
   ├── 冷冻水供水温度设定下调：7°C → 6°C
   ├── 利用早晨低温尽量预冷建筑蓄热体
   └── AHU 全速运行

3. 白天运行
   ├── 冷站全台运行（N台冷机满负荷）
   ├── 冷冻水泵满频运行
   ├── 冷却塔风机满频运行
   ├── AHU 送风温度设定下调 1°C
   ├── 监控重点：
   │   ├── 冷冻水供回水温差（应维持 5°C）
   │   ├── 冷却水温度（高温日冷却塔散热效率下降）
   │   ├── 冷机负载率（接近 100% 时预警）
   │   └── 室内温度分布（西晒区域重点关注）
   └── 应急措施：
       ├── 如冷量仍不足 → 适当提高室内温度设定（26→27°C）
       ├── 通知租户临时调高温度设定
       └── 非核心区域（如地下车库通风）降频运行，释放电力

4. 下午峰值（14:00~17:00）
   ├── 峰值负荷可达设计冷量的 110~120%
   ├── 密切监控电力需量（避免超容量罚款）
   ├── 错峰关闭非必要电力负载
   └── 冷站运行参数实时优化

5. 傍晚恢复
   ├── 室外温度下降后逐步减机
   └── 恢复正常排程
```

**判断逻辑**:
- 室外温度 > 38°C：启动极端高温预案
- 室外温度 > 40°C：启动最高级别保障
- 冷站总负载 > 90%：发送容量预警
- 冷却水出水温度 > 35°C：加大冷却塔运行力度

**预期结果**:
- 室内温度不超过 27°C（可接受范围）
- 无设备因过载跳闸
- 电力需量不超合同容量

### 场景5：冷机故障应急

**触发条件**: 冷水机组报故障停机 (CH_Status = FAULT)

**涉及设备**: 故障冷机 + 备用冷机 + 相关水泵 + 末端

**操作步骤**:

```
T+0s   故障检测：
       ├── BMS 接收冷机故障信号
       ├── 生成紧急告警（推送至值班工程师/运维主管/远程中心）
       └── 记录故障时间、故障码

T+30s  自动应急响应：
       ├── 自动启动备用冷机（如有空闲冷机）
       ├── 对应冷冻水泵/冷却水泵联动切换
       ├── 故障冷机相关阀门自动关闭（隔离）
       └── 如无备用冷机 → 进入降容运行

T+2min 系统稳定检查：
       ├── 确认备用冷机正常运行
       ├── 冷冻水供水温度恢复正常
       ├── 管网压差正常
       └── 末端供冷恢复

T+5min 容量评估：
       ├── 当前冷机台数能否满足负荷？
       ├── 如不足（高负荷时段）：
       │   ├── 适当提高送风温度设定（节省冷量）
       │   ├── 非核心区域 FCU 降速/关闭
       │   ├── 通知租户临时调高设定温度
       │   └── 安排维修人员紧急检修故障冷机
       └── 如足够（低负荷时段）：
           └── 维持运行，安排非紧急检修

T+15min 通报：
        ├── 推送故障简报给物业经理
        ├── 如影响租户 → 通知受影响租户
        └── 联系冷机厂商技术支持（如需）

T+2~24h 修复：
         ├── 维修人员到场检修
         ├── 更换故障部件
         ├── 冷机复位试运行
         ├── 确认恢复正常后纳入自动调度
         └── 编写故障报告
```

**判断逻辑**:
- 单台故障 + 有备用 → 自动切换，工单处理
- 单台故障 + 无备用 + 低负荷 → 降容运行
- 单台故障 + 无备用 + 高负荷 → 应急降级 + 紧急维修
- 多台故障 → 最高级别应急，启动全楼应急预案

**预期结果**:
- 备用冷机 2min 内启动
- 室内温度波动 < 2°C
- 故障 4h 内修复（常见故障）或 24h 内修复（重大故障）

### 场景7：制冷/制热季节切换

**触发条件**: 气象数据 + 室外温度趋势达到切换阈值

**涉及设备**: 冷站 + 热源 + AHU + FCU + 管网阀门

**操作步骤**:

```
切换前准备（提前 1~2 周）：
├── 设备检查
│   ├── 目标系统设备试运行（如冬→夏：冷机试运行）
│   ├── 阀门动作测试（冷热水切换阀）
│   ├── 传感器校准
│   └── 管网充水/排气/冲洗
│
├── 过渡方案制定
│   ├── 确定切换日期（参考气象预报）
│   ├── 通知租户
│   └── 安排运维人员值守

正式切换（以制冷→制热为例）：
├── 步骤 1：冷站停机
│   ├── 冷机停止
│   ├── 冷冻水泵/冷却水泵停止
│   ├── 冷却塔排空、防冻处理
│   └── 冷站阀门关闭
│
├── 步骤 2：管网切换（四管制可同时供冷热，二管制需切换）
│   ├── 二管制：排空冷冻水 → 注入热水 → 排气
│   └── 四管制：关闭冷水阀 → 开启热水阀
│
├── 步骤 3：热源启动
│   ├── 锅炉/热泵点火试运行
│   ├── 热水循环泵启动
│   └── 系统升温至设定温度
│
├── 步骤 4：末端切换
│   ├── AHU 控制逻辑切换（制热模式）
│   ├── FCU 温控器切换至制热
│   └── VAV 再热功能启用（如有）
│
└── 步骤 5：验证
    ├── 各楼层温度达标
    ├── 无异常告警
    └── 切换完成报告

过渡季特殊处理：
├── 上午需供暖、下午需制冷的情况
├── 可能需要"晨暖午冷"混合模式
└── 四管制系统优势明显，二管制需依赖运营经验判断
```

**判断逻辑**:
- 四管制系统：无需完整切换，可同时供冷热
- 二管制系统：需完整切换流程，通常 1~2 天完成
- 过渡季（3~4 月、10~11 月）：灵活应对，可能频繁调整

**预期结果**:
- 切换期间室内温度波动 < 3°C
- 零投诉（提前通知租户）
- 切换后系统运行稳定

- 租户可通过 APP 自助控制

## 5. 能效指标（KPI）

### 5.1 冷站级 KPI

| KPI 名称 | 英文 | 计算公式 | 优秀 | 良好 | 一般 | 差 | 单位 |
|---------|------|---------|------|------|------|---|------|
| 冷机 COP | Chiller COP | 制冷量(kW) / 输入功率(kW) | >6.0 | 5.0~6.0 | 4.0~5.0 | <4.0 | - |
| 冷机 EER | Chiller EER | COP × 3.412 | >20.5 | 17~20.5 | 13.6~17 | <13.6 | BTU/W·h |
| 冷站综合 COP | Plant COP | 总制冷量 / (冷机+水泵+冷却塔总功率) | >5.0 | 4.0~5.0 | 3.0~4.0 | <3.0 | - |
| 单位冷量能耗 | kW/RT | (冷机+水泵+冷却塔总功率) / 总制冷量(RT) | <0.65 | 0.65~0.85 | 0.85~1.1 | >1.1 | kW/RT |

| KPI 名称 | 英文 | 计算公式 | 优秀 | 良好 | 一般 | 差 | 单位 |
|---------|------|---------|------|------|------|---|------|
| 单位面积空调费用 | HVAC Cost/m² | HVAC 年运营成本 / 建筑面积 | <30 | 30~50 | 50~80 | >80 | 元/m²·年 |
| 冷站度电冷价 | Cost/RT·h | 冷站电费 / 总制冷量(RT·h) | <0.8 | 0.8~1.2 | 1.2~1.6 | >1.6 | 元/RT·h |
| 节能率 | Energy Saving | (基线能耗-实际能耗) / 基线能耗 | >20% | 10~20% | 5~10% | <5% | % |

### 5.4 KPI 计算公式详解

#### 冷站综合 COP

```
Plant COP = Q_cooling / P_total

其中：
  Q_cooling = 冷冻水流量(m³/h) × 4.186(kJ/kg·°C) × ΔT(°C) × 1000/3600
            = 冷冻水流量 × 1.163 × ΔT  (kW)

  P_total   = P_chiller + P_CHWP + P_CWP + P_CT_fan

  ΔT = CHWR_Temp - CHWS_Temp (冷冻水供回水温差)
```

#### kW/RT

```
kW/RT = P_total / Q_cooling_RT

其中：
  Q_cooling_RT = Q_cooling(kW) / 3.517
  1 RT (冷吨) = 3.517 kW

  P_total = P_chiller + P_CHWP + P_CWP + P_CT_fan (kW)
```

#### HVAC EUI

```
HVAC EUI = E_hvac / A

其中：
  E_hvac = 年度 HVAC 总电耗 (kWh)
         = 冷站 + AHU/MAU 风机 + FCU 风机 + 热源(如有) + 其他辅助
  A = 建筑总面积 (m²)
```

## 6. 租户空调计费

### 6.1 计费模式

#### 模式一：按面积分摊

```
月费 = 租赁面积 × 单价(元/m²/月) × 季节系数

特点：
  - 最简单，适合物业管理费中包含空调费的情况
  - 无论使用多少都一样，不激励节能
  - 适用：乙级写字楼、小面积租户

示例：
  租赁面积 500m²，单价 8 元/m²/月
  月费 = 500 × 8 = 4,000 元
```

#### 模式二：按冷量计量

```
月费 = 冷量表读数差(kWh) × 冷量单价(元/kWh) + 基本费(如有)

特点：
  - 最公平，多用多付
  - 需要安装冷量表（BTU meter），成本较高
  - 激励租户节能
  - 适用：甲级写字楼、大面积租户

示例：
  月冷量消耗 8,000 kWh，单价 0.35 元/kWh
  月费 = 8,000 × 0.35 = 2,800 元
```

#### 模式三：按时间计费

```
月费 = 正常时段费(含在物业费) + 延时空调时间 × 延时费率

特点：
  - 正常工作时间空调费含在物业管理费中
  - 仅对延时/加班空调额外收费
  - 适用：工作时间统一的写字楼

示例：
  物业费已含工作日 8:00~18:00 空调
  加班 10 次 × 3h × 200 元/h = 6,000 元/月
```

#### 模式四：混合计费

```
月费 = 基本费(按面积) + 超量费(按冷量) + 延时费(按时间)

特点：
  - 基本费保底 + 超量部分按实际计量
  - 兼顾公平性和管理便利性
  - 适用：甲级写字楼

示例：
  基本费：500m² × 5 元 = 2,500 元
  超量费：超出基准冷量部分 × 0.4 元/kWh
  延时费：加班时段另计
```

### 6.2 延时空调费率

| 时段类型 | 费率参考 | 说明 |
|---------|---------|------|
| 工作日延时（18:00~22:00） | 150~300 元/h/层 | 最常见的加班时段 |
| 工作日深夜（22:00~次日 8:00） | 200~400 元/h/层 | 需额外安保 |
| 周末白天（8:00~18:00） | 200~350 元/h/层 | 冷站从零启动 |
| 周末夜间 | 250~450 元/h/层 | 冷站+安保+额外人工 |
| 法定节假日 | 300~500 元/h/层 | 最高费率 |

> 注：以上为按楼层计费参考，也可按面积或按区域计费。具体费率因城市、楼宇等级而异。

### 6.3 分户计量方案

```
方案一：冷量表分户计量
├── 每个租户安装独立冷量表（供回水管温度传感器 + 流量计）
├── BMS 自动抄表（间隔 15min~1h）
├── 月度出账：冷量读数 × 单价
├── 优点：最准确、最公平
└── 缺点：冷量表成本高（5000~15000 元/台）、需定期校准

方案二：电量分户计量
├── 各租户 FCU/VAV 独立电表
├── 加权系数分摊冷站公共能耗
├── 月度出账：(租户末端电量 + 公摊冷站电量) × 电价
├── 优点：成本较低、电表便宜
└── 缺点：FCU 电量不等于冷量，精确度一般

方案三：时间 + 面积分摊
├── 按运行时间和面积加权分摊
├── 无需额外安装计量设备
├── 月度出账：总空调费 × (租户面积×运行时长) / Σ(所有租户面积×运行时长)
├── 优点：无额外硬件成本
└── 缺点：精确度最低，大小租户不公平

方案四：混合方案（推荐）
├── 大面积租户（>500m²）：冷量表计量
├── 小面积租户（<500m²）：电量计量或面积分摊
├── 公共区域：按面积公摊
└── 延时空调：独立按时间计费
```
